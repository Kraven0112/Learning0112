Event loupe is asynchronous operation in javscript.

call stack is mechanism in javascript provided by web browser to track the functions which is running and which can be executed first.

setTimeout(function(){}) and setTimeout(function xyz(){}) the main difference between these two ways are reuse of function.

If we use anonymous function we cannot reuse it anywhere, because it will be specified for where it is declared.

If we use named function we can reuse it anywhere according to its scope.

So the use of event loop is to push the operation from callback queue to call-stack

All the promise related tasks are set in microtask queue.

callback queue is also known as task queue.

There are two types of task queue :> microtask queue and macrotask queue.

microtask operations are like promises,mutation observers.

macrotask operations are like setTimeout,setInterval,DOM events,Network requests

In event loop microtasks get more priority than macrotasks while performing asynchronous operations.

All the callback functions which come through promise are related to microtask-queue.

microtasks have higher priority than macrotasks.
